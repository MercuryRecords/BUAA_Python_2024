## 题目创建

- 请求方式：POST
- 请求URL：/api/problem_create
- 请求参数：

| 参数名                   | 参数说明   | 是否必须  | 类型  | 例子                        | 备注                          |
|-----------------------|--------|-------|-----|---------------------------|-----------------------------|
| username              | 用户名    | 是     | 字符串 | "JohnDoe"                 |                             |
| problem_group_creator | 问题组创建者 | 是     | 字符串 | "JohnDoe"                 |                             |
| problem_group_title   | 问题组标题  | 是     | 字符串 | "Math Questions"          | 长度不超过50个字符                  |
| type                  | 题目类型   | 是     | 字符串 | "c" 或 "b"                 | "c" 表示选择题，"b" 表示填空题         |
| content               | 题目内容   | 是     | 字符串 | "What is the sum of 1+1?" | 长度不超过1000个字符                |
| ans_count             | 答案数量   | 是     | 整数  | 1                         | 选择题答案数量，填空题答案数量不超过7个        |
| answer                | 答案     | 选择题必填 | 字符串 | "A"                       | 选择题答案，大写字母，A、B、C等           |
| field1                | 填空题答案1 | 填空题必填 | 字符串 | "1"                       | 填空题答案，长度不超过100个字符           |
| field2                | 填空题答案2 | 填空题必填 | 字符串 | "2"                       | 填空题答案，长度不超过100个字符           |
| field3                | 填空题答案3 | 填空题必填 | 字符串 | ""                        | 填空题答案，长度不超过100个字符，无答案时传空字符串 |
| field4                | 填空题答案4 | 填空题必填 | 字符串 | ""                        | 填空题答案，长度不超过100个字符，无答案时传空字符串 |
| field5                | 填空题答案5 | 填空题必填 | 字符串 | ""                        | 填空题答案，长度不超过100个字符，无答案时传空字符串 |
| field6                | 填空题答案6 | 填空题必填 | 字符串 | ""                        | 填空题答案，长度不超过100个字符，无答案时传空字符串 |
| field7                | 填空题答案7 | 填空题必填 | 字符串 | ""                        | 填空题答案，长度不超过100个字符，无答案时传空字符串 |

- 返回信息：

| 状态码 | 信息     | 数据                                 | 备注 |
|-----|--------|------------------------------------|----|
| 200 | 创建成功   | {"code": 200, "message": "创建成功"}   | -  |
| ... | ...    | ...                                | -  |

这个接口用于在指定问题组下创建题目。需要提供问题组的创建者和标题来唯一标识问题组。题目类型分为选择题（"c"）和填空题（"b"），根据题目类型不同，需要提供的答案和填空题答案字段不同。创建成功后，返回创建成功的提示。

## 题目顺序调整

- 请求方式：POST
- 请求URL：/api/problem_adjust_order
- 请求参数：

| 参数名                   | 参数说明   | 是否必须 | 类型  | 例子               | 备注                       |
|-----------------------|--------|------|-----|------------------|--------------------------|
| username              | 用户名    | 是    | 字符串 | "JohnDoe"        |                          |
| problem_group_creator | 问题组创建者 | 是    | 字符串 | "JohnDoe"        |                          |
| problem_group_title   | 问题组标题  | 是    | 字符串 | "Math Questions" | 长度不超过50个字符               |
| index                 | 题目索引   | 是    | 整数  | 1                | 问题组中题目的索引，从1开始           |
| newindex              | 新的题目索引 | 是    | 整数  | 1                | 新的题目索引，范围在 1 到 问题组最大索引之间 |

- 返回信息：

| 状态码 | 信息     | 数据                                 | 备注 |
|-----|--------|------------------------------------|----|
| 200 | 调整成功   | {"code": 200, "message": "调整成功"}   | -  |
| ... | ...    | ...                                | -  |

这个接口用于调整指定问题组下题目的顺序。需要提供问题组的创建者、标题和题目索引来唯一标识题目，以及新的题目索引 `newindex`。题目上传者和问题组拥有者可以调整题目的顺序。调整成功后，返回调整成功的提示。

注意：在调整题目顺序时，需要更新问题组中其他题目的索引值，以保持题目顺序的一致性。新的题目索引值必须在 1 到 问题组最大索引之间。
